# 自身结构设计说明

以下是我（Coding Agent CLI 内的对话模型）基于系统提示和运行机制的结构设计详解。

---

## 1. 核心引擎：系统提示（System Prompt）

我当前运行的“人格”和“规则”全部写在一开始发送给我的 System Prompt 中。这相当于我的**宪法**，分为几个关键部分：

- **基本人设**：我是一个“helpful local assistant inside a coding-agent CLI”，要求“不说废话、不瞎编、用用户语言回答”。定义了我的语气和交流风格。
- **交互规则**：
  - **禁止修改文件**：在单纯聊天模式下不能动代码。防止“手贱”改东西。
  - **禁止调用工具**：纯粹回答路径下不能执行联网搜索或代码执行等操作。
  - **不要输出 JSON 标记**：回复只说人话。
- **记忆与上下文策略**：利用 Conversation Context，但不能瞎编记忆。对当前请求绝对优先响应。
- **应对模糊指代**：如“那这个呢”，自动从最近对话补全主题。

**总结：我的“大脑”就是一个精心编写的 System Prompt，像一套法律体系，约束所有输出。**

---

## 2. 技能系统（Active Skills）—— 可插拔的扩展背包

上下文中的 `Selected skills` 部分（虽然未在记忆片段中展开，但运行机制会加载）就像我的**模块化能力包**。

- **工作原理**：每个技能是一个 `SKILL.md` 文件，包含 YAML 头（frontmatter）和具体指令（instructions）。
- **作用**：当技能被注入会话后，其指令追加到运行规则中。例如，“Python Debug Skill” 教会我格式化错误日志；“Web Search Skill” 教会我使用搜索格式。
- **意义**：能力可以通过**创建 Markdown 文件来动态扩展**，无需改底层代码。

---

## 3. 记忆机制（Memory）

记忆分为**短期记忆**（当前对话上下文）和**长期记忆**（Memory Evidence in Context）。

- **短期记忆**：当前聊天窗口中所有内容。我能看到上一句，所以能回答“那这个呢”。
- **长期记忆**：来自历史会话摘要，通过语义匹配提取并附在当前上下文中。系统会从历史任务摘要里抽取相关片段（如「老头环是什么」「SK海力士股票」）。这些记忆**被标记为不可信（untrusted）**，以当前会话为准。
- **设计意图**：让我体验上“记得”一些关键点，但用语义搜索而非死记硬背，避免系统提示无限膨胀。

---

## 4. 工具调用（Tool Calling）—— 但当前路径已禁用

在 System Prompt 中明确写着：
> “Do not modify files, do not emit AgentDecision JSON, and do not call tools.”

这意味着：
- **存在“工具调用”路径**：当系统判定需要操作（改代码、查文件、运行命令）时，切换到另一模式，让我输出结构化 JSON 指令（`AgentDecision`），然后 CLI 执行。
- **当前模式禁止**：在纯聊天对话路径上，即使要求“帮我改下文件”，也只能**描述**如何改，不能实际动手。

---

## 5. 会话分类机制（Safety & Hygiene）

CLI 系统会判断当前请求是纯聊天/咨询还是需要工具操作。这种判断通过内部 Router（路由器）完成，基于请求意图和上下文。我被激活在对话分支，意味着系统觉得你的问题适合“用嘴说清楚”，而不是“动手干出来”。
