# Plan 模式（规划模式）

mini-coding-agent 提供有限的只读 **plan 模式**。在此模式下，代理仅能使用只读工具（如搜索代码、读取文件、搜索网络），**不能**修改文件或执行命令。这适合先规划复杂任务，经你确认后再执行。

## 如何进入 Plan 模式

在交互式 CLI 中：
- 使用命令 `/plan` 进入规划模式（此时如果带了任务描述，会立即规划）
- 直接输入 `mini-agent plan "描述你的任务"` 从非交互模式进入

## Plan 模式下的操作

进入后，你可以向代理描述任务，它会：
1. 使用只读工具分析仓库
2. 生成一个分步计划（步骤列表）
3. 在终端输出计划，**不会**直接执行

注意：plan 模式下所有写工具（apply_patch、run_command）都会被拒绝。

## 退出 Plan 模式并执行

当你对计划满意时：
- 使用 `/plan off` 退出规划模式（计划仍保留在当前会话中）
- 然后使用 `/execute` 执行最后一份成功计划

你也可以直接退出规划模式后手动执行其他命令。

## 状态保存

Plan 状态会随 Session 保存。即使中断会话，恢复后续操作。

## 示例

```
$ mini-agent plan "为项目添加 TypeScript 类型定义文件"
[plan] 1. 读取现有代码结构
[plan] 2. 分析需要添加的类型
[plan] 3. 创建 .d.ts 文件并导出接口
[plan] 4. 验证类型检查通过
$ /plan off
$ /execute
```
